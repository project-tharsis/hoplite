# WK v1.2 Feature Enrichment 规格

日期：2026-05-20
状态：待实现
受众：实现代理

## 1. 背景

WK v1.2 predicate mining enhancement 已经完成。

当前事实：

- b-003 prompt blind spots 在 clean subset 上达标。
- WK v1.2 rule distillation pipeline 可以稳定运行。
- enhanced predicate mining 评估了 3591 个 predicate。
- implementation candidates = 0。
- exploratory candidates = 0。
- replay passes = false。
- summary decision = `collect_more_features`。

这不是继续调门槛的问题。

当前 features 主要是 box score 摘要：

- 胜平负、比分、控球、射门、xG、传球、角球、犯规、换人时间。

但 B 的很多判断依赖比赛过程：

- 谁先进球。
- 阿森纳领先后有没有丢掉控制。
- 丢球发生在什么比分状态。
- 换人时阿森纳领先、落后还是僵持。
- 换人后比分状态有没有改善。
- xG 优势是否被浪费。
- 对手射门少但质量高，还是射门多但质量低。
- 75 分钟后是否守住领先。

所以本阶段要补充过程型 features。

目标是让下一轮 WK/B 分歧可以被 deterministic predicate 捕捉。

## 2. 产品目标

建立 `features_version = "v2"` 的过程型特征层，使系统能够：

1. 从现有 raw match JSON / analyze report 中提取比赛过程特征。
2. 把新 features 写入 102 条 feature-backed KB entries。
3. 让 prompt 能展示新 features，触发 b-004 重新评估。
4. 让 predicate search space 使用新 features 重新挖 WK v1.2 候选规则。
5. 保持保守闭环：不改 WK 规则，不降低 replay 门槛，不静默覆盖 B/human。

本阶段的成功标准不是一定产出 WK v1.2 implementation rule。

成功标准是：

- features v2 可复现、可回放、可审计。
- b-004 能基于新 features 生成高质量评估。
- 新一轮 mining 要么产出 candidate，要么给出更精确的 feature gap。

## 3. 非目标

- 不修改 `src/labels/weak_labeler.py` 的信号规则。
- 不降低 candidate 门槛。
- 不把 prompt blind spot 直接变成 WK rule。
- 不把 B evaluation 冒充 human review。
- 不使用 embedding、训练模型、外部数据库或在线服务。
- 不从比分编造事件。
- 不从 narrative 文本反推事件。
- 不重跑 b-002。
- 不做 JSON 到数据库迁移。

## 4. 核心原则

### 4.1 只提取，不判断

`FeatureExtractor` 只负责确定性事实提取。

不要在 extractor 中写类似“控制好/差”的战术判断。

判断应留给：

- weak labeler
- prompt builder
- predicate mining
- evaluator B

### 4.2 缺失要显式表达

如果 raw/report 中没有足够信息：

- 数值字段使用 `None`。
- 列表字段使用 `[]`。
- 布尔派生字段在 search space 中为 `false`。
- `missing_data` 加入明确字段名。

禁止用默认值伪装成事实。

### 4.3 过程特征必须可复现

同一份 raw match JSON 多次运行，features 必须完全一致。

排序规则必须稳定：

- goal events 按 `(minute, team, player)` 排序。
- substitution events 按 `(minute, player)` 排序。

### 4.4 feature schema 变化必须 bump version

新增字段会改变 prompt、B evaluation stale 判断和 predicate mining 输入。

因此本阶段必须：

- 引入 `FEATURES_VERSION = "v2"`。
- `prepare_evaluation` 输出 `features_version: "v2"`。
- `save_evaluation` / backfill / self-iteration metadata 使用 v2。
- `make-jobs --only stale-evaluation` 能识别 v1 → v2。

## 7. Feature Schema v2

在 `src/features/extractor.py` 的 `MatchFeatures` 中新增字段。

### 7.1 Goal Timing / Score State

新增字段：

```python
match_duration_minutes: int = 90

arsenal_goal_minutes: list[int] = field(default_factory=list)
opponent_goal_minutes: list[int] = field(default_factory=list)

first_goal_minute: Optional[int] = None
first_goal_team: Optional[str] = None          # "arsenal" | "opponent" | None
arsenal_scored_first: Optional[bool] = None
arsenal_conceded_first: Optional[bool] = None

arsenal_led_at_any_point: bool = False
arsenal_trailed_at_any_point: bool = False
lead_change_count: int = 0

minutes_leading: Optional[int] = None
minutes_trailing: Optional[int] = None
minutes_level: Optional[int] = None

goals_for_after_75: int = 0
goals_against_after_75: int = 0
```

语义：

- `match_duration_minutes = max(90, max(event.minute))`。
- 如果有比分但 goal events 不完整，时间类字段为 `None` 或 `[]`，并在 `missing_data` 中加入 `goal_events`。
- `minutes_leading/trailing/level` 由 `score_state_timeline` 分段计算。
- `lead_change_count` 只统计领先方从 Arsenal 到 opponent 或 opponent 到 Arsenal 的变化，不把 level 计为领先方。

### 7.2 Lead Protection

新增字段：

```python
max_lead: int = 0
max_deficit: int = 0

lead_lost_count: int = 0
equalizers_conceded_after_leading: int = 0
goals_conceded_while_leading: int = 0

first_lead_minute: Optional[int] = None
last_lead_minute: Optional[int] = None
final_state_from_first_lead: str = "never_led"   # protected | lost | never_led

led_after_75: bool = False
late_lead_lost: bool = False
late_lead_protected: bool = False
late_goals_conceded_while_leading: int = 0
```

语义：

- `max_lead` 是 Arsenal 最大领先球数。
- `max_deficit` 是 Arsenal 最大落后球数，用正数表示落后幅度。
- `lead_lost_count` 统计从 Arsenal 领先变为平局或落后的次数。
- `equalizers_conceded_after_leading` 统计 Arsenal 领先时被追平的对手进球。
- `final_state_from_first_lead = protected` 当 Arsenal 曾领先且终场仍赢。
- `final_state_from_first_lead = lost` 当 Arsenal 曾领先但终场未赢。
- `led_after_75 = true` 当 75 分钟或之后任一分段 Arsenal 处于领先。
- `late_lead_lost = true` 当 75 分钟后 Arsenal 领先变为平局或落后。
- `late_lead_protected = true` 当 75 分钟后 Arsenal 领先且终场赢。

### 7.3 xG Conversion / Overperformance

新增字段：

```python
xg_overperformance_for: Optional[float] = None       # arsenal_goals - xg_for
xg_overperformance_against: Optional[float] = None   # opponent_goals - xg_against
xg_result_gap: Optional[float] = None                # score_margin - xg_delta

dominant_xg_no_win: bool = False
won_without_xg_edge: bool = False
```

语义：

- 只有 `xg_for`、`xg_against`、`xg_delta` 都存在时计算。
- `dominant_xg_no_win = xg_delta >= 1.0 and result != "W"`。
- `won_without_xg_edge = result == "W" and xg_delta < 0`。
- xG 缺失时，这些 Optional 字段为 `None`，布尔字段为 `False`，`missing_data` 保持现有 `xG`。

### 7.4 Substitution Effect by Score State

扩展 `substitution_windows`。

每条 substitution window 新增：

```json
{
  "minute": 66,
  "player": "Trossard",
  "scored_after": true,
  "score_margin_at_sub": 0,
  "score_state_at_sub": "level",
  "goals_for_after_sub": 1,
  "goals_against_after_sub": 0,
  "net_goals_after_sub": 1
}
```

新增 top-level 字段：

```python
first_sub_minute: Optional[int] = None
first_sub_score_margin: Optional[int] = None
first_sub_score_state: Optional[str] = None      # leading | level | trailing | unknown

subbed_while_leading: bool = False
subbed_while_level: bool = False
subbed_while_trailing: bool = False

goals_for_after_first_sub: int = 0
goals_against_after_first_sub: int = 0
net_goals_after_first_sub: int = 0
```

语义：

- `score_margin_at_sub` 使用 substitution minute 之前最近的 score state。
- 如果 substitution 与 goal 同分钟，先按事件原始顺序；无法确定顺序时，使用该分钟之前的 score state，并在 window 加 `same_minute_order_uncertain: true`。
- 没有 substitutions 时，top-level minute/state 为 `None`，布尔为 `False`，goals after 为 0。
- 如果 score timeline 缺失，state 为 `unknown`，并在 `missing_data` 加 `score_state_at_substitution`。

### 7.5 Opponent Shot Quality Pressure

新增字段：

```python
arsenal_xg_per_shot: Optional[float] = None
opponent_xg_per_shot: Optional[float] = None
arsenal_xg_per_shot_on_target: Optional[float] = None
opponent_xg_per_shot_on_target: Optional[float] = None

opponent_high_quality_chances: bool = False
opponent_low_volume_high_quality: bool = False
```

语义：

- `xg_per_shot = xg / shots`，要求 xG 和 shots 都存在且 shots > 0。
- `xg_per_shot_on_target = xg / shots_on_target`，要求 xG 和 shots_on_target 都存在且 shots_on_target > 0。
- `opponent_high_quality_chances = opponent_xg_per_shot >= 0.12 or opponent_xg_per_shot_on_target >= 0.35`。
- `opponent_low_volume_high_quality = shots_against <= 8 and opponent_high_quality_chances`。
- 缺失时 Optional 字段为 `None`，布尔字段为 `False`。

## 14. Phase 7：Enriched Predicate Search Space

扩展 `scripts/self_iterate.py` 的 `_derive_feature_view()`。

新增 deterministic predicate features：

```text
scored_first                  = arsenal_scored_first is true
conceded_first                = arsenal_conceded_first is true
trailed_then_won_or_drew       = arsenal_trailed_at_any_point and result != L
led_but_failed_to_win          = arsenal_led_at_any_point and result != W
protected_lead                = final_state_from_first_lead == protected
lost_lead                     = final_state_from_first_lead == lost
late_lead_lost                = late_lead_lost is true
late_lead_protected           = late_lead_protected is true
dominant_xg_no_win            = dominant_xg_no_win is true
won_without_xg_edge           = won_without_xg_edge is true
bad_xg_conversion             = xg_overperformance_for <= -0.75
opponent_clinical_finishing   = xg_overperformance_against >= 0.75
opponent_high_quality_chances = opponent_high_quality_chances is true
sub_positive_from_level       = first_sub_score_state == level and net_goals_after_first_sub > 0
sub_negative_while_leading    = first_sub_score_state == leading and net_goals_after_first_sub < 0
sub_no_change_while_trailing  = first_sub_score_state == trailing and net_goals_after_first_sub <= 0
late_game_control_loss        = led_after_75 and late_lead_lost
```

要求：

- 每个新增 predicate 在 `feature_definitions` 记录 required inputs。
- 缺失 input 时 predicate 为 `false`，并写入 row-level `missing_features`。
- 不枚举 arbitrary numeric threshold；阈值必须在本 spec 固定。

验收：

- `enriched_feature_search_space.json` 包含旧 predicate + 新 predicate。
- rows 数量等于 b-004 clean subset。
- `missing_features` 不出现别名误报，例如 `shots_delta`。
